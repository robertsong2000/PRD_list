# 个人健身与减重网站 PRD

## 1. 产品概述

### 1.1 产品定位
一个综合性的个人健身与减重管理平台，通过智能化的健身活动追踪、减重进度监测、个性化健身计划生成、营养摄入记录以及激励机制，帮助用户实现科学的健身减重目标。平台支持与主流健身追踪设备集成，提供数据可视化和智能分析，打造全方位的健康管理体验。

### 1.2 目标用户画像

#### 核心用户群体

**用户画像一：白领减重人群**
- **年龄**: 25-40岁
- **职业**: 办公室白领、企业管理人员
- **痛点**: 久坐不动、工作压力大、缺乏运动时间、饮食不规律
- **目标**: 减重5-15kg、改善体质、建立健康生活方式
- **行为特征**: 移动端使用为主、碎片化时间操作、需要激励和提醒

**用户画像二：健身爱好者**
- **年龄**: 18-35岁
- **特征**: 有一定健身基础、追求身材优化
- **痛点**: 训练计划不系统、营养知识不足、进度难以量化
- **目标**: 增肌减脂、提升体能、优化训练效果
- **行为特征**: 数据驱动、使用专业设备、愿意分享成果

**用户画像三：健康恢复人群**
- **年龄**: 30-55岁
- **特征**: 医生建议需要控制体重和运动
- **痛点**: 缺乏专业指导、担心运动损伤、需要监测指标
- **目标**: 安全减重、改善健康指标、建立长期习惯
- **行为特征**: 谨慎保守、重视数据准确性、需要专业建议

**用户画像四：产后恢复女性**
- **年龄**: 25-35岁
- **特征**: 产后需要恢复身材和体能
- **痛点**: 时间碎片化、体能下降、需要科学指导
- **目标**: 恢复孕前体重、重塑体型、提升体能
- **行为特征**: 需要灵活时间安排、重视产后安全、社群支持需求强

### 1.3 核心价值主张

#### 对用户的价值
- **科学追踪**: 精准记录健身活动和身体数据变化
- **个性化方案**: 基于AI算法生成定制化的健身和营养计划
- **实时反馈**: 即时查看进度和成就，保持动力
- **智能提醒**: 运动提醒、饮食记录、目标达成提醒
- **社交激励**: 成果分享、排行榜、挑战活动

#### 对产品的价值
- **高留存率**: 通过数据累积和社交功能提升用户粘性
- **高参与度**: 每日打卡、记录和查看进度形成使用习惯
- **可扩展性**: 模块化设计便于功能迭代和扩展
- **数据资产**: 用户健康数据为AI优化和产品改进提供基础

## 2. 核心功能需求

### 2.1 功能模块总览

#### 2.1.1 用户管理模块
- 用户注册与登录（邮箱/手机号/第三方登录）
- 个人信息管理（基础信息、身体数据、目标设定）
- 账户安全设置（密码、隐私设置、数据导出）
- 会员订阅管理

#### 2.1.2 身体数据追踪模块
- 体重记录与趋势分析
- 体脂率、肌肉量等身体成分追踪
- 身围测量记录（腰围、臀围、胸围等）
- BMI计算与历史曲线
- 进度照片管理
- 健康指标监测（血压、心率、血氧等）

#### 2.1.3 健身活动追踪模块
- **手动记录**
  - 有氧运动（跑步、游泳、骑行等）
  - 力量训练（器械、自由重量）
  - 球类运动、瑜伽、普拉提等
  - 自定义运动类型

- **自动同步**
  - 支持主流健身设备集成
  - 智能穿戴设备数据同步
  - 手机传感器数据采集
  - GPS轨迹记录

- **活动详情**
  - 运动时长、距离、配速
  - 消耗卡路里计算
  - 心率区间分析
  - 运动强度评估

#### 2.1.4 个性化健身计划模块
- **计划生成**
  - 基于用户目标和身体状况的AI推荐
  - 减重计划、增肌计划、塑形计划
  - 初级、中级、高级难度分级
  - 时间周期选择（4周/8周/12周）

- **计划内容**
  - 每周训练安排（训练日、休息日）
  - 具体训练动作和组数、次数
  - 动作视频演示和文字说明
  - 强度递进方案
  - 替代动作选项

- **计划管理**
  - 计划预览与调整
  - 训练完成度跟踪
  - 计划暂停与恢复
  - 计划效果评估

#### 2.1.5 营养摄入记录模块
- **食物记录**
  - 食物数据库搜索（含常见食物和包装食品）
  - 手动添加自定义食物
  - 菜谱和套餐快捷记录
  - 扫码识别包装食品
  - 语音输入食物

- **营养分析**
  - 热量计算与目标对比
  - 三大营养素比例（碳水、蛋白质、脂肪）
  - 微量营养素追踪
  - 每餐营养分布
  - 营养趋势报告

- **饮食建议**
  - 基于目标的个性化热量建议
  - 营养素摄入优化建议
  - 健康食谱推荐
  - 饮食习惯分析

#### 2.1.6 数据可视化模块
- **仪表盘**
  - 今日概览（运动、饮食、体重）
  - 周度/月度统计图表
  - 目标达成进度条
  - 快捷操作入口

- **趋势图表**
  - 体重变化曲线
  - 运动量趋势
  - 热量摄入/消耗对比
  - 身体成分变化
  - 多时间段对比（周、月、季、年）

- **报告生成**
  - 周报、月报自动生成
  - PDF报告导出
  - 数据对比分析
  - 成就总结

#### 2.1.7 激励系统模块
- **成就体系**
  - 连续打卡成就（7天、30天、100天）
  - 运动里程碑（累计里程、总时长）
  - 减重成就（每减重1kg、达成目标）
  - 特殊挑战完成徽章

- **积分奖励**
  - 完成训练获得积分
  - 记录饮食获得积分
  - 邀请好友获得积分
  - 积分兑换会员/周边

- **排行榜**
  - 减重进度榜
  - 运动时长榜
  - 连续打卡榜
  - 好友排行榜

- **挑战活动**
  - 平台官方挑战（如"30天减重挑战"）
  - 用户自定义挑战
  - 小组挑战赛
  - 季度主题挑战

#### 2.1.8 社交互动模块
- **社区功能**
  - 动态发布（文字、图片、视频）
  - 关注与粉丝系统
  - 内容点赞、评论、分享
  - 话题标签

- **好友系统**
  - 好友添加与管理
  - 好友动态查看
  - 互相点赞和鼓励
  - 私信功能

- **内容分享**
  - 成果分享到社交平台
  - 训练计划分享
  - 健康报告分享
  - 分享海报生成

### 2.2 功能优先级

#### P0 - 必须有（MVP核心功能）
- 用户注册登录
- 身体数据记录（体重、BMI）
- 手动记录健身活动
- 基础饮食记录（热量计算）
- 数据可视化（体重趋势、运动统计）

#### P1 - 应该有（核心增强功能）

- 详细的营养分析
- 成就徽章系统
- 健身设备集成
- 数据报告导出

#### P2 - 可以有（扩展功能）
- 社交社区功能
- 排行榜与挑战活动
- 高级数据分析
- 语音记录食物
- 多端数据同步优化

#### P3 - 未来考虑（创新功能）
- AI健身教练
- AR动作指导
- 基因检测集成
- 线上课程市场
- 健身直播功能

## 3. 用户流程设计

### 3.1 新用户引导流程

```
用户首次访问
    ↓
注册/登录
    ↓
欢迎引导页（产品介绍）
    ↓
填写个人信息
  - 性别、年龄、身高、当前体重
  - 健身经验（无经验/初级/中级/高级）
  - 可用运动设备
  - 运动偏好（有氧/力量/综合）
    ↓
设定目标
  - 目标类型（减重/增肌/塑形/保持）
  - 目标数值（目标体重、体脂率等）
  - 目标时间（4周/8周/12周/自定义）
    ↓
生成个性化计划
  - 展示AI生成的健身计划
  - 展示每日热量建议
  - 展示预期目标达成曲线
    ↓
计划确认与开始
  - 调整计划选项
  - 设置训练提醒
  - 完成引导，进入主页
```

### 3.2 日常使用流程

#### 晨间流程
1. 查看今日计划（首页仪表盘）
2. 晨起体重记录（如适用）
3. 查看昨日数据总结
4. 接收今日训练提醒

#### 训练流程
```
打开训练页面
    ↓
查看今日训练计划
    ↓
选择训练项目
    ↓
查看动作演示
    ↓
开始训练
  - 记录组数和次数
  - 记录使用的重量
  - 记录主观感受
    ↓
完成训练
  - 查看训练总结
  - 自动记录到活动日志
  - 更新训练进度
  - 获得积分/成就
```

#### 饮食记录流程
```
打开饮食记录页面
    ↓
选择餐次（早餐/午餐/晚餐/加餐）
    ↓
记录食物
  - 搜索食物数据库
  - 或扫描包装食品
  - 或手动输入
  - 或选择常用食物
    ↓
添加份量
  - 选择份量单位
  - 或输入克数
    ↓
确认添加
  - 查看营养信息
  - 查看剩余热量配额
  - 继续添加或完成记录
```

### 3.3 设备集成流程

```
进入设备设置页面
    ↓
选择设备品牌
  - Apple Watch
  - Fitbit
  - Garmin
  - 小米手环
  - 华为手环
  - 其他
    ↓
授权连接
  - 跳转第三方授权页面
  - 用户同意授权
    ↓
数据同步
  - 历史数据导入
  - 设置自动同步频率
    ↓
同步完成
  - 查看导入的数据
  - 设置数据展示偏好
```

### 3.4 进度回顾流程

```
进入进度页面
    ↓
选择时间维度
  - 本周/本月/本季/本年
  - 或自定义时间段
    ↓
查看多维度数据
  - 体重变化曲线
  - 运动量统计
  - 饮食趋势
  - 身体成分变化
    ↓
生成报告
  - 自动总结亮点
  - 对比目标进度
  - AI分析与建议
    ↓
分享或导出
  - 分享到社交平台
  - 导出PDF报告
  - 保存到相册
```

## 4. 用户界面规范

### 4.1 设计原则

#### 视觉风格
- **简洁现代**: 扁平化设计，减少视觉干扰
- **积极向上**: 使用明亮色彩和动态效果传递活力
- **数据清晰**: 图表和信息可视化清晰易读
- **情感化设计**: 通过动画和交互给予积极反馈

#### 色彩系统
- **主色调**: 活力橙色（#FF6B35）- 代表能量和行动
- **辅助色**: 蓝绿色（#00B4A6）- 代表健康和平衡
- **背景色**: 浅灰白（#F5F7FA）- 护眼舒适
- **文字色**: 深灰（#2C3E50）- 保证可读性
- **成功色**: 绿色（#27AE60）- 目标达成
- **警示色**: 红色（#E74C3C）- 偏离目标提醒

#### 字体系统
- **中文**: 思源黑体 / 苹方
- **英文**: San Francisco / Roboto
- **数字**: DIN / SF Mono（数据展示）

## 6. 数据安全与隐私

### 6.1 数据安全要求

#### 6.1.1 数据加密
- **传输加密**: 全站 HTTPS/TLS 1.3
- **存储加密**:
  - 敏感数据（密码、支付信息）使用 AES-256 加密
  - 数据库透明加密 (TDE)
  - 备份数据加密存储

#### 6.1.2 访问控制
- **身份认证**:
  - 支持多因素认证 (MFA)
  - JWT Token 认证
  - OAuth 2.0 第三方登录
  - Token 自动刷新和过期机制

- **权限管理**:
  - 基于角色的访问控制 (RBAC)
  - API 接口权限校验
  - 数据访问权限隔离

#### 6.1.3 数据隐私
- **个人信息保护**:
  - 符合 GDPR、CCPA 等隐私法规
  - 用户数据匿名化选项
  - 数据最小化收集原则

- **健康数据保护**:
  - 符合 HIPAA 标准（如涉及美国用户）
  - 健康数据特殊加密存储
  - 明确的健康数据使用授权

- **用户权利**:
  - 数据访问权
  - 数据更正权
  - 数据删除权（被遗忘权）
  - 数据导出权（可移植性）
  - 撤回同意权

#### 6.1.4 安全审计
- **操作日志**:
  - 用户操作日志记录
  - 管理员操作审计日志
  - API 调用日志
  - 异常行为检测

- **合规性**:
  - 定期安全扫描
  - 渗透测试
  - 安全漏洞修复 SLA

### 6.2 数据备份与恢复

#### 备份策略
- **数据库备份**:
  - 每日全量自动备份
  - 每小时增量备份
  - 跨地域异地备份

### 12.3 卡路里计算公式

#### 基础代谢率 (BMR)
**Mifflin-St Jeor公式**:
- 男性: BMR = 10 × 体重 + 6.25 × 身高 - 5 × 年龄 + 5
- 女性: BMR = 10 × 体重 + 6.25 × 身高 - 5 × 年龄 - 161

#### 每日总能量消耗 (TDEE)
TDEE = BMR × 活动系数

活动系数:
- 久坐少动: 1.2
- 轻度活动: 1.375
- 中度活动: 1.55
- 高度活动: 1.725
- 极度活动: 1.9

#### 运动消耗计算
消耗热量 = MET × 体重 × 时间
